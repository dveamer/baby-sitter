# Campaign Ops Checklist

권한 없이 미리 준비할 수 있는 운영 체크리스트입니다.

## Before launch

* 1차 국가를 `인도네시아 / 필리핀 / 브라질`로 확정했는지 확인
* 각 국가별 landing URL을 `landing-links.md` 기준으로 준비
* Play Console 문구를 `play-console-paste-ready.md` 기준으로 준비
* 숏폼 3종의 훅/자막 언어를 확정
* 커뮤니티 관리자 승인 요청 문구를 언어별로 복사해둠
* tester용 안전 안내를 `tester-safe-setup-guide.md` 기준으로 준비
* 문의 이메일과 답변 템플릿 준비
* 무료 코드/체험 정책 초안 준비

## Launch week

* 각 채널별 링크에 `utm_source`, `utm_medium`, `utm_campaign`, `utm_content` 값을 넣음
* 첫 10개 문의는 어떤 질문이 많은지 따로 기록
* 첫 10명 설치자 기준으로 setup confusion 포인트를 메모
* `QR 연결`, `자장가 녹음`, `첫 모니터링 시작`까지 막히는 지점 분류
* 국가별로 스크린샷/첫 문장 반응이 다른지 확인

## Weekly review

* 스토어 방문 대비 설치율
* 설치 대비 첫 설정 완료율
* 첫 설정 완료 대비 QR 연결 성공률
* QR 연결 대비 첫 모니터링 시작률
* 체험 신청 대비 실제 피드백 회수율
* 문의 중 가장 자주 반복된 오해 3개

## If setup friction is high

* 설치 전 기대치와 실제 기능이 어긋나는지 확인
* `same Wi-Fi only` 설명이 늦게 나오는지 확인
* 최소 준비물이 명확히 보이는지 확인
* viewer가 브라우저라는 점이 충분히 이해되는지 확인

## If community response is weak

* 모집글이 너무 광고처럼 보이는지 확인
* `free access code + feedback request` 구조로 다시 정리
* 첫 문장에서 `old Android phone` 가치가 바로 보이는지 확인
* 안전 문구가 너무 약하거나 너무 무거워서 이탈시키는지 확인

## Go / no-go decision after 2 weeks

* `go`: 첫 설정 완료율과 피드백 응답률이 둘 다 살아있는 국가
* `hold`: 설치는 나오지만 setup drop이 큰 국가
* `stop`: 문의/반응/설정 데이터가 모두 약한 국가
