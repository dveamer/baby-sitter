## 작업 디렉토리 스크립트

등급 기준 원문 확인에 사용한 JS 파일은 이 디렉토리에 둔다.

- `extract_naver_member_levels.js`: `naver-cafe-tasks.md`에서 `원문 확인 필요` 행을 골라 Chrome 디버깅 포트(`9222`)에 연결하고, 각 `member-level` 페이지의 등급 DOM 테이블을 읽어 `naver_member_levels.json`을 만든다.
- `extract_naver_member_levels_extra.js`: 이미 확인됨으로 적힌 기존 행 중 재확인이 필요한 번호를 골라 `naver_member_levels_extra.json`을 만든다. 기본 대상은 `1, 2, 3, 27, 31`.

- `update_naver_tasks_from_levels.js`: `naver_member_levels.json` 기준으로 해당 행을 `확인됨`으로 갱신한다.
- `update_naver_tasks_all_verified.js`: `naver_member_levels.json`과 `naver_member_levels_extra.json`을 합쳐 전체 확인 행을 원문 기준으로 갱신한다.

## openChrome 원문 확인 절차

네이버 `member-level` 페이지는 일반 `curl`로는 앱 shell만 내려오고 실제 등급 표는 JS 렌더링 뒤에 보인다. 원문 확인은 별도 Chrome 디버깅 프로필을 열어 DOM 테이블을 직접 읽는 방식으로 진행한다.

```bash
open -na "Google Chrome" --args \
  --remote-debugging-port=9222 \
  --user-data-dir=/private/tmp/codex-naver-openchrome \
  --no-first-run \
  --no-default-browser-check \
  https://cafe.naver.com/ca-fe/cafes/13584633/member-level
```

그 다음 작업 디렉토리에서 필요한 추출/갱신 스크립트를 실행한다.

```bash
node extract_naver_member_levels.js
node extract_naver_member_levels_extra.js
node update_naver_tasks_all_verified.js
```

추출 기준:
- `table.tbl_role tbody tr`의 두 번째 행을 "두 번째 등급"으로 본다.
- 등급명은 `th`, 설명은 `td p`, 자동/신청 조건은 `td li`에서 읽는다.
- 조건 문구의 `자동으로 등업`과 `등업 게시판에 등업 신청` 여부를 구분해 기록한다.

참고 API:
- member-level DOM은 내부적으로 `https://apis.naver.com/cafe-web/cafe-mobile/CafeMemberLevelInfo?cafeId={cafeId}` JSON을 호출한다.
- 캐시 복원 시 `memberLevelList`의 두 번째 항목을 "두 번째 등급"으로 본다.
- `leveluptype` 값은 현재 확인 기준으로 `2 = 자동 등업`, `3 = 등업게시판 신청 가능`, 그 외는 별도 수치 조건 없음으로 기록한다.
